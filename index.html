About

Trained in printmaking, drawn into code by accident.

I started in a print studio, setting type by hand and learning to like the parts of a process I couldn't fully control. A residency in 2015 put me in a room with generative artists for the first time, and I never really left — the loom and the loop turned out to be the same idea.

Today my studio takes on a handful of commissions a year alongside self-directed work, and runs a small seasonal workshop teaching working artists to write their first rule-based piece. I care most about work that stays legible as a system — where you can feel the rule even as the output surprises you.

How a piece gets made

Four passes, in order

Every commission moves through the same sequence — the order matters, since each pass constrains the next.

01

Source the constraint

Find the dataset, material, or input signal that will actually drive the piece — weather, sound, sensor, archive. The constraint comes first; the visuals follow from it.

02

Sketch the rule set

Write the smallest set of rules that could plausibly produce something interesting, and run it far too many times to see where it breaks.

03

Tune by hand

Adjust weights, palettes, and thresholds until the system's own tendencies start to show through — this pass is judgment, not automation.

04

Let it run

Ship the system, not a single output — into a gallery, a browser, or a print run — and let it keep generating past the point I stop watching.
